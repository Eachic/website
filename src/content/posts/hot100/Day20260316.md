# Day20260316

## 除了自身以外数组的乘积

给你一个整数数组 nums，返回 数组 answer ，其中 answer[i] 等于 nums 中除了 nums[i] 之外其余各元素的乘积 。

题目数据 保证 数组 nums之中任意元素的全部前缀元素和后缀的乘积都在  32 位 整数范围内。

请 不要使用除法，且在 O(n) 时间复杂度内完成此题


### 题解

就是一个简单的前后缀和的问题

```c++

class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        vector<int> pre,las;
        int len = nums.size();
        pre.resize(len);
        las.resize(len);
        pre[0] = 1;
        for(int i=1;i<len;i++){
            pre[i] = nums[i-1] * pre[i-1];
        }
        las[len-1] = 1;
        for(int i=len-2;i>=0;i--){
            las[i] = nums[i+1] * las[i+1];
        }

        vector<int> ans;
        ans.resize(len);
        for(int i=0;i<len;i++){
            ans[i] = pre[i] * las[i];
        }

        return ans;
    }
};
```

## 缺失的第一个正数

给你一个未排序的整数数组 nums ，请你找出其中没有出现的最小的正整数。

请你实现时间复杂度为 O(n) 并且只使用常数级别额外空间的解决方案。


###  题解

该题对时间复杂度与空间复杂度都有很高的要求。本质是对原数组空间信息的利用最大化。
该题目最容易想到的方法就是利用哈希表。但是哈希表要额外的空间复杂度，但是可以把原数组改造成一个哈希表。

首先应该考虑输出范围，一定是在1~N+1 中。然后我们遍历数组，对于数组中不在1~N+1 的值，这些值没有提供任何有用信息，因此可以全部赋值为一个相同的数，考虑N+1。然后我们再次遍历数组，如果该数处于1~N+1 中，我们可以把数组nums[x-1]的数赋值为负数，用此表示该数出现过。最后第三次逆序遍历数组，求出1~N+1中没出现的最小数。

```c++
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        for(int i=0;i<nums.size();i++){
            if(nums[i] <=0 || nums[i] > nums.size()){
                nums[i] = nums.size()+1;
            }
        }

        for(int i=0;i<nums.size();i++){
           if(abs(nums[i]) >=1 && abs(nums[i]) <= nums.size()){
              nums[abs(nums[i]) - 1]  = -abs(nums[abs(nums[i])-1]);
           }
        }
        int ans = nums.size() +1;
        for(int i=nums.size()-1;i>=0;i--){
            if(nums[i] <= 0) continue;
            ans = i+1;
        }

        return  ans;
    }
};
```
